# 坦克大战 — 产品设计文档 v2.0

---

## 1. 产品概述

经典 Battle City（坦克大战）的 Python + Pygame 复刻版。双阵营对战，玩家操控绿色坦克守卫己方基地，击退 AI 红色坦克并摧毁敌方基地即可获胜。

### 核心玩法定位

> 以 NES 原版《Battle City》为蓝本，保留核心体验，用现代工程方式重构。

---

## 2. 核心玩法规则

### 2.1 地图系统

- **网格大小**：26×26 逻辑格子（半格粒度），每格 24px → 624×624 像素
- **图块类型**：

| 图块 | 坦克通过 | 子弹通过 | 可摧毁 | 说明 |
|------|:-------:|:--------:|:------:|------|
| 空地 | ✅ | ✅ | - | |
| 砖墙 | ❌ | ❌（子弹摧毁砖块） | ✅ | **半格精度**：每个 2×2 半格，子弹只摧毁命中的那一半 |
| 钢铁 | ❌ | ❌（子弹消失） | ❌ | 仅玩家满级（3 星）可摧毁 |
| 水域 | ❌ | ✅ | - | 子弹飞过，坦克沉没 |
| 冰面 | ✅（打滑） | ✅ | - | 坦克惯性滑动 |
| 森林 | ✅ | ✅ | - | 坦克被遮蔽（视觉上层绘制） |
| 基地 | ❌ | ✅（命中即结束） | ❌ | 双方各有一个，被摧毁则游戏结束 |

### 2.2 阵营设定

| | A 方（AI/红方） | B 方（玩家/绿方） |
|--|---------------|-----------------|
| 坦克数量 | 20 辆/关 | 1 辆 |
| 同时在场 | ≤4 辆 | 1 辆 |
| 生命 | 无限（补充至满编） | 3 条命 |
| 出生点 | 顶部 3 个固定点 | 底部中央 |
| 基地 | 顶部 | 底部 |

### 2.3 坦克属性

| 属性 | 玩家 | AI 普通 | AI 快速 | AI 重甲 | AI 精英 |
|------|------|---------|---------|---------|---------|
| 速度 | 1格/次 | 1格/次 | 2格/次 | 1格/次 | 1格/次 |
| 血量 | 1 | 1 | 1 | 4 | 1 |
| 射击间隔 | 15帧 | 30帧 | 20帧 | 40帧 | 25帧 |
| 掉落道具 | - | 10% | 10% | 30% | 100% |
| 外观 | 绿色 | 灰色 | 蓝色 | 银色 | 红色(闪烁) |

### 2.4 胜负条件

- **玩家胜利**：摧毁全部 20 辆 AI 坦克 **或** 摧毁 AI 方基地
- **玩家失败**：3 条命用完 **或** 己方基地被摧毁
- **平局**：不存在（总有一方胜利）

## 4. AI 行为设计

### 4.1 行为状态机

```
                   ┌──────────────┐
     ┌─────────────┤  巡视(Roam)  ├──────────────┐
     │             └──────┬───────┘              │
     │                    │                      │
     ▼                    ▼                      ▼
┌──────────┐       ┌────────────┐         ┌──────────┐
│ 追击玩家  │       │  推进基地  │         │  卡住换向 │
│ (Attack)  │       │ (PushBase) │         │ (Stuck)  │
└─────┬─────┘       └─────┬──────┘         └─────┬────┘
      │                   │                      │
      └───────────────────┴──────────────────────┘
```

### 4.2 状态切换逻辑（优先级从高到低）

1. **追击（Attack）**：与玩家同行/同列且无遮挡时 → 瞄准玩家方向
2. **推进（PushBase）**：与敌方基地同行/同列且无遮挡时 → 瞄准基地方向
3. **巡视（Roam）**：默认状态，随机移动+变向
4. **卡住换向（Stuck）**：连续 N 帧未移动 → 换方向

### 4.3 路径选择增强

- 当挡在面前的是砖墙 → 有 60% 概率朝墙方向射击（开路）
- 被其他坦克阻挡 → 等 30 帧后换向绕行
- 优先选择**通往玩家或基地的方向**（加权随机，而非完全随机）

## 5. 道具系统（Power-Up）

### 5.1 道具列表

| 图标 | 名称 | 效果 |
|------|------|------|
| ⭐ | 星星 | 升级武器：1星→双发 2星→快速弹 3星→可摧毁钢铁 |
| 🛡️ | 头盔 | 10 秒无敌护盾 |
| ⏱️ | 时钟 | 冻结所有 AI 坦克 10 秒 |
| 💣 | 炸弹 | 消灭屏幕上所有 AI 坦克 |
| 🧱 | 铁铲 | 基地围墙升级为钢铁，持续 20 秒 |
| ❤️ | 坦克 | 额外一条命 |

### 5.2 掉落规则

- AI 坦克被摧毁时有概率掉落道具（不同 AI 类型概率不同）
- 红色闪烁的 AI 精英坦克 **100% 掉落**
- 道具出现在被摧毁坦克的位置
- 玩家坦克经过即可拾取
- 15 秒未被拾取则消失

## 7. 技术架构

### 7.2 技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| 语言 | Python 3.10+ | |
| 渲染 | Pygame 2.x | |
| 碰撞检测 | `pygame.sprite.Group` + `groupcollide()` | 批量碰撞，C 层优化 |
| 实体系统 | `pygame.sprite.Sprite` 继承 | 统一生命周期管理 |
| 状态管理 | 状态模式（State Pattern） | 类替代 if-else 分支 |
| 关卡数据 | JSON + TXT | 易于编辑和扩展 |
| 资源 | 像素风 SpriteSheet（PNG） | 而非 `pygame.draw` 图元 |
| 音效 | WAV | 射击、爆炸、拾取等 |

### 7.3 核心循环（伪代码）

```python
class Game:
    def run(self):
        while True:
            dt = self.clock.tick(60)
            events = pygame.event.get()
            self.state.handle_events(events)
            self.state.update(dt)
            self.state.draw(self.screen)
            pygame.display.flip()
```

### 7.4 碰撞检测策略

```
每帧检测:
  1. 子弹 vs 砖墙 → 销毁砖块（半格粒度），销毁子弹
  2. 子弹 vs 钢铁 → 销毁子弹（满级子弹可摧毁钢铁）
  3. 子弹 vs 坦克 → 友军跳过，敌军扣血/销毁
  4. 子弹 vs 基地 → 游戏结束
  5. 坦克 vs 地图 → 阻止移动到不可通行图块
  6. 坦克 vs 坦克 → 阻止重叠
  7. 坦克 vs 道具 → 拾取道具
  
使用 pygame.sprite.groupcollide() 批量处理
```

## 8. 视觉风格

### 8.1 美术风格

- **风格**：16-bit 像素风（致敬经典 NES，但更高清）
- **调色板**：每色 8bit（256 色限制），明亮对比色
- **分辨率**：窗口 832×768（26 格 × 24px 地图 + 右侧 208px HUD 面板）
